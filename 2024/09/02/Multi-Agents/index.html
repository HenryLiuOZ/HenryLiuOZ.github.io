Chapter3：FRC Three simple

- FRC比赛规则


  1. 场景内允许两只队伍，每队三个机器人在场地内比赛，机器人通过将NOTE投入到 己方场地内的 Speaker中以得分
  2. 比赛前15秒，机器人处于自动阶段，按照预先设定的程序得分；在剩下的2分15秒内，操作手控制自己的机器人。机器人从Source处收集来NOTEs，并送入Speaker和AMP中得分。每当一个队伍在其AMP中积累2个音符，操作者可以决定激活 放大状态 的时间。在放大状态的10秒内投入Speaker可以获得更高得分
  3. 如果双方队伍都在手动阶段的前45秒内按下 合作竞争按钮 重新利用一个音符（将投入AMP的NOTE取出用于得分），那么比赛中的所有队伍都会获得1个合作竞争分（这会影响他们在比赛中的排名），同时旋律奖励（一种得分成就，当一个队伍在扩音器和扬声器中积累了一定数量的音符时，可以获得额外的分数）所需的音符数量也会减少
  4. 机器人被允许跨过半场以干扰对手的得分（如争夺对方的NOTE，迫使对方犯规等），并可以与对手发生接触（但接触时间需要小于5秒）

Chapter4：FRCOne collection

在MALiR环境陷入阻滞后，我决定探索一些很早就遇到的，在单智能体RL中疑惑不解的问题。
最主要的问题是奖励延迟问题，在一次很早的汇报中，我将这个问题描述为由于智能体做出正确动作（shoot）到进球并获得奖励之间有大约2s的延迟，当智能体得到奖励时，智能体在执行其他动作。直观地，智能体会在做任意动作时获得奖励，这导致智能体无法学习。

然而，在这里我犯了一个直觉性的错误。根据贝尔曼方程定义，状态价值包含了未来轨迹的discounted return：

简单来说，假设智能体在St=s做出了正确动作a（为了严谨，假设这是获得环境奖励的唯一状态，动作对），导致在第t+k步获得奖励r，那么$q(s,a)=\gamma^k*r$

真正会导致价值网络无法训练的是部分可观测问题POMDP。$q(s,a)=r+\gamma*E(G_{t+1}|S_t=s)$，前面我们提到奖励延迟，r=0，v(s’)=r，其中s’是s的后继状态。在这个投篮任务中，显然应该是v(投篮后2s)具有高价值，而如果代理的观测空间有限(ie，只能捕捉当前的位置)，那么它无法区分 St = [(x,y) && shot=T] 与 St = [(x,y) && shot=F] 。环境的部分可观测性在这里造成了状态混淆state aliasing，这导致代理训练困难。解决方法是构建更完善的观测空间，也常用RNN嵌入一段长度的历史状态作为观测输入

Chapter5：MALiR —— Multi-Agent LLM instructed Reinforcement Learning

- idea

受ellm的启发，我在环境中实现了OracleFRC和LLMFRC代理。首先回顾一下ellm：

ellm是一个将large language models应用到intrinsiclly motivated RL的算法。简单来说，具有人类先验知识的大语言模型在每一步为环境代理生成建议，而环境代理在达成建议目标时将得到内在奖励，从而环境中代理只有达成一串长序列动作才能获得的稀疏奖励被优化为了频繁奖励

类似的，在特定时间步(间隔为5秒)，LLM收集当前同队三个代理的状态与观测(textform)并为每个代理生成建议，在下一次建议更新前，达成建议目标的代理将得到个体奖励。在这里，我要求LLM生成suggested position（对应代理的二维坐标）和suggested action（对应shoot/amp/intake/drop四个动作），并且代理必须先达到目标位置再完成动作才能获得奖励

在MALiR架构中加入分层式强化学习框架是受到Learning Multi-Agent Loco-Manipulation for Long-Horizon Quadrupedal Pushing的启发。在这个长距离木箱推动任务中，RRT根据机器狗木箱当前位置，目标位置和障碍物情况生成一段轨迹，高层控制器根据目标轨迹优化生成子目标，中层控制器只完成机器狗推动木箱到达子目标的任务，这是将长距离任务分解为短距离任务

在MALiR架构中引入分层式框架的主要目的是扁平化不同动作的序列长度，简单来说，在不使用分层的框架中，ask for note需要的动作序列仅为1，intake和shoot需要需要2-3步，而amp和pass这样的动作显然需要相当长的动作序列。将这些不同动作抽象为子任务，并且预训练执行这些子任务的executor。对于Finetuner而言，所有这些子任务的动作序列都为1。其次，可以通过策略层输出的子任务评估代理当前的策略

- task definition

the task we consider here is a decentralized-POMDP defined by: (N,S,O,A,P,r,γ) where N ≥ 1 is the number of agents and S is the state space of the environment. O is the joint observation space of all agents O := O1 × … × ON where Oi is the observation space of agent i. At time t, the environment is in state st ∈ S and oi_t ∈ Oi is the local observation of agent i which is correlated with st. The environment state contains more information than the observations available to the agents. A is the joint action space of all agents A := A1 × … × AN where Ai is the action space of agent i.
P: S×A×S →[0,1] is the transition function where P(s′|s, a) is the probability that the environment transitions to state s′ given the current state s and joint action a ∈ A. r: S×A →R is the shared reward function where r(s a) is the reward received by all agents when the joint action a ∈ A is taken and the environment is in state s ∈ S.

- method

MALiR的策略层是一个CTDE的MARL算法，Finetuner: Oi -> Gi 接收智能体的观测并为执行层生成目标位置和目标动作。在训练阶段，所有智能体同时做出决策并request LLM 为每个智能体生成建议。计算Fintuner goals与LLM suggestions的余弦相似度，相似度高的Fintuner会得到intrinsic reward，从而加速Fintuner收敛到具有人类先验知识的策略。同时，Finetuner接收环境的group reward，从而收敛到最大化环境奖励。值得注意的是，在当前的FRCThree训练环境中， 我并没有将两队的奖励设计为标准的混合竞争模式(红队得分会造成蓝队的group reward减少) 。这样做的目的是可以通过tensorboard中的group reward趋势图直观评估代理的策略

执行层Executor: Oi, Gi -> Ai 是一个单智能体RL算法。为了与LLM生成的建议对齐，或者说FRC这个类篮球游戏本就需要顺序性动作，Executor被指定优先达到目标位置，再做出目标动作。为了环境的稳定性，Executor和Finetuner分开进行训练

- problem and result

传统的分层式强化学习的high-level RL的decision时机是low-level RL goal achieved，这是由于对于go to and shoot这样的顺序性任务，如果是以固定频率更新低层控制器会一直忙于前往新的目标点而无法做出得分动作（这里我没录视频，效果是环境中的6个机器人一直在东奔西跑，而几乎无法做出得分动作）

而如果令high-level RL在goal achieved时更新decision，在MARL的框架下，也会带来问题。CTDE的MARL算法拥有中心化的价值网络，它的输入是环境所有智能体的观测，为了保证价值网络能够训练，所有Finetuner的策略更新必须是同步的，而每个机器人有不同的目标，有的目标位置远有的目标位置近，这导致Finetuner的策略难以同步更新。

在查阅了一些文献后，我发现我遇到的是MAHRL领域的一个general problem。这篇文章指出分层策略的使用（对于所有代理）也引入了一些MAHRL独有的附加问题，如 跨不同代理同步子任务终止 。不同agent的决策同步受到其子任务的终止的影响。可以通过在任何一个代理的子任务终止时强制终止所有代理的子任务，或引入等待时间，直到所有子任务自然终止，以同步方式终止所有代理的子任务。该方案易于协调决策，但也在终止条件上施加了强制偏差，这可能导致次优行为。另一种方案是异步终止，其中每个代理的子任务独立于其他代理的子任务的终止而终止。该方案没有任何强制中断，但由于决策是异步发生的，因此难以协调决策。

一个可行的解决方案是在任一执行层goal achieved时软更新所有Finetuner的策略（要求所有Finetuner返回策略，但只有达成目标的执行层更新决策）

但可惜这个方法并没有救活我的MALiR，策略层可以趋近Oracle输出的建议目标，但始终无法有效得分。做到这里我停手了，再进一步需要探究MAHRL领域的内容，而这与我的最初目标不吻合


Chapter6：Reflections

首先是反思MALiR的分层结构带来的一些问题。虽然它令智能体的策略更直观了，但

  1. 分层结构对执行层的要求很高，有较大随机行为或得分效率不理想的执行器显然无法使Finetuner收敛到有效的策略。智能体需要做出截然不同的动作，而只是基于其巨大的观测空间中的几个变量（suggested_position, suggested_action, position_successed, action_successed），为了训练出理想的随机行为更少的执行器，这里我做了非常多的尝试。使用LLM引导RL学习的本意是减少人工设计奖励的困难，但是在设计执行器这一步却需要精细的奖励设计
  2. 分层结构增大了策略层的解空间，同时降低了策略层的决策频率，这反而增大了最终算法训练所需的时间。对比AgentFRC,其输出大小是3+3+3+5（前后+左右+转向+4个动作），Finetuner的输出大小是94*44+5（目标点坐标+4个动作）。AgentFRC的决策频率约为10/sec，而Finetuner的决策频率低于1/sec

其次是反思ELLM架构的有效性。之前介绍过ELLM本质上是在CB-IM RL的架构上修改。在CB-IM RL中，内在奖励Rint(o,a,o′|g)定义为代理动作带来的环境改变了与goal之间的余弦相似度，这个目标通常是手工定义的子任务，而ELLM将手工定义的子任务替换为了LLM生成的建议。同样属于Intrinsically Motivated RL的算法是好奇心内在奖励，智能体在遇到新颖的状态时受到reward。简单来说，好奇心内在奖励激励代理学习完整的环境模型。基于预测误差的好奇心奖励训练一个预测模型来预测智能体从状态st采取动作at的下一状态 $\hat{s_{t+1}}$，与真实的下一状态 $s_{t+1}$做比较，奖励代理到达具有更高预测误差的环境 $R_{int}=\alpha*||\hat{s_{t+1}}-s_{t+1}||^2$ 。也有基于状态访问频率，潜在空间距离，信息增益的好奇心奖励。

mlagents自带有好奇心奖励，可以看到curiosity的效果事实上比ELLM的方法更好（均采用lastAction观测）。我认为好奇心内在奖励是一种更为普适性的解决稀疏奖励的方法
